4//! # Trusted Platform Module 2.0 (TPM2) Structures and Commands
5//!
6//! This base crate provides:
7//!   - Definitions of the TPM2 constants and structures.
8//!   - Definitions of the [TPM2 Commands](commands).
9//!   - Common traits for [`Marshal`]ing and [`Unmarshal`]ing.
10//!
11//! ## Design Goals
12//!
13//! This crate defines a low-level interface to any TPM2. The types and
14//! commands in this crate can be used to either communicate with an existing
15//! TPM2 (i.e., be used in a client) or to _implement_ a TPM2.
16//!
17//! Many types in this crate have a direct counterpart in "Part 2: Structures"
18//! of the [TPM2 Specification]. Types that map 1:1 to the specification have a
19//! `Tpm` prefix. For example:
20//!   - The [`TpmtHa`] enum corresponds to the `TPMT_HA` type.
21//!   - The [`TpmiAlgHash`] C-like enum corresponds to the `TPMI_ALG_HASH` type.
22//!
23//! Conversely, types or items that either do not map to a type in the spec
24//! (e.g., [`Marshal`] or [`Command`]) or have semantics differing from those in
25//! the spec (e.g., [`Alg`]) will not have a `Tpm` prefix.
26//!
27//! will not have a `Tpm` prefix.
28//!
29//! [TPM2 Specification]: https://trustedcomputinggroup.org/work-groups/trusted-platform-module/
30//!
31//! ## Platform Support
32//!
33//! Unlike some other crates under the TPM-RS project, this crate is intended
34//! to work on platforms and in environments which lack the Rust Standard
35//! Library or memory allocation. To that end, this crate is `#[no_std]`,
36//! and does not use the `std` or `alloc` libraries (only `core` is used).
37//!
38//! ## Panics
39//!
40//! Furthermore, we **strive to avoid panics in this library**. While this cannot
41//! be statically guaranteed by Rust, we will run tests to ensure that panic code
42//! is not emitted, provided sufficient optimizations are enabled.
43//!
44//! ## Dependencies
45//!
46//! To allow this crate to be used in constrained environments (like kernels or
47//! TPM2 implementations), we disallow any _runtime_ dependencies. Also, we
48//! restrict our [build-dependencies] to a subset necessary to create Procedural
49//! Macros (`proc_macro`, `syn`, `quote`, etc...). We will have more
50//! [dev-dependencies] for running additional tests, but such additional
51//! dev-dependencies should be gated by opt-in Cargo features.
52//!
53//! [build-dependencies]: https://doc.rust-lang.org/cargo/reference/specifying-dependencies.html#build-dependencies
54//! [dev-dependencies]: https://doc.rust-lang.org/cargo/reference/specifying-dependencies.html#development-dependencies
55#![no_std]

120impl MarshalFixed for Alg {
121    const SIZE: usize = 2;
122    type Array = [u8; 2];
123    #[inline(always)]
124    fn marshal_fixed(&self, arr: &mut [u8; Self::SIZE]) {
125        self.0.marshal_fixed(arr)
126    }
127}
128impl UnmarshalFixed for Alg {
129    #[inline(always)]
130    fn unmarshal_fixed(&mut self, arr: &[u8; Self::SIZE]) -> Result<(), UnmarshalError> {
131        self.0.unmarshal_fixed(arr)
132    }
133}
